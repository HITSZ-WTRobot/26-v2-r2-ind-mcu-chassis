# 麦轮底盘轨迹规划

## 区域描述：

Zone1: x = [8.00, 9.44], y = [1.23, 6.00] 除 Zone2 的矩形区域

Zone2: x = [9.30, 10.80], y = [4.50, 6.00] 的矩形区域

Zone3: x = [9.50, 12.00], y = [0.20, 6.00] 除 Zone2 的矩形区域

## 障碍：

Zone 1,2,3 以外的区域

x = [9.30, 10.80], y = [4.47, 4.53] 的矩形区域

## 底盘描述

底盘为一个 X 方向长 0.76cm，Y 方向宽 0.52cm 的矩形。考虑碰撞时，应膨胀 3cm

## 需求：

离线生成 3 条高质量的轨迹，不在意离线生成时间，以时间最短为优化目标

每个轨迹点由 8 个 float 构成：[x, y, yaw, h, dx, dy, dyaw, dh]

从 3 个不同的起点到达同一个终点位置。

起点：

1. [8.43, 1.8, 0deg, 0.412, 0, 0, 0, 0]
2. [8.43, 3.0, 0deg, 0.412, 0, 0, 0, 0]
3. [8.43, 4.2, 0deg, 0.412, 0, 0, 0, 0]

终点：

1. [11.25, 2.5, -90deg, 0.412, 0, 0, 0, 0]

## 限制：

1. 当底盘矩形部分任意位置落入 Zone2 时，认为其位于 Zone2. 底盘在 Zone2 中必须具有 yaw = -90deg 或 0deg, h = 0.3
2. 底盘约束必须符合 /home/syhanjin/workspace/robocon2026/master/path_mecanum4 中的变换关系和轮速/加速约束，轮速限制 160rad/s, 轮加速度限制 60rad/s^2
3. 膨胀底盘区域完全不能碰到障碍物，这是硬约束
4. x, y, yaw 三轴必须联合优化；h 轴可以独立优化
5. 满足速度和位置的积分关系

## 环境：

使用 uv，和全局 uv 缓存

## 输出：

500Hz 轨迹 .hpp，按曲线命名空间导出，并在每个头文件内包含 `kSampleHz`、`kDt`、`kPointCount`、`kDuration` 和轨迹数组
三条轨迹的 gif，包括：移动动画 用矩形描述车状态；车高度变化图；该时刻车体三轴速度/加速度；该时刻四轮速度/加速度
