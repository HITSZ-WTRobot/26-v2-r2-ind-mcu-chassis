# CHANGELOG

## 2026-06

### 新增
- **离线轨迹系统**：新增 `OfflineTrajectoryFollower` 4-DOF 离线轨迹跟随器，支持编译期静态轨迹点(100Hz)通过 Slave 控制器回放；支持 X 轴镜像翻转
- **降采样工具**：新增 constexpr 轨迹降采样器，将 500Hz 原始轨迹转为 100Hz
- **planning monorepo**：新增离线轨迹规划子项目 `planning/`，基于 CasADi 优化器生成时间最优轨迹

### 修复
- 修复离线轨迹模式与主动模式的正确切换
- 增大升降跟随器的限幅
- 修复离线轨迹导出错误
- 修复反馈帧可能无法恢复的问题
- 修复线程安全问题，弃用不安全的原上台阶协议 (`0x30-0x34`)
- 修复架在高台阶发送上低台阶指令时收腿导致翻车的问题
- 修复先取高台阶再下低台阶的撞车问题
- 修复 grip 取矛动作执行顺序问题

### 变更
- **台阶命令重构**：移除不安全的距离式上/下台阶命令 (`0x30-0x34`)，统一使用姿态式 `0x50..0x5F` 台阶命令组
- **Step 命令队列**：为 Step 增加环形缓冲区队列，允许连续发送指令，队列自动排序执行，有重复指令保护
- **上位机断联保护**：增加上位机断联检测，超时后自动急停
- **R1 台阶动作**：新增 `0x35 StepUpR1` 和 `0x36 StepUpR1Direct` 命令；为上 R1 动作单独降速处理
- **升降精度**：判断升降结束时要求 1cm 的升降精度
- 更新 EKF 中 lidar 的标准差
- 更新底盘控制器为 PIDPD 并更新参数
- 微调 spear grab 动作参数

## 2026-05

### 新增
- **红外接收模块**：通过 USART6 DMA 接收红外协议(`0xA0..0xA3`)，触发对接释放动作
- **Grip 姿态命令**：新增 `0x16 SetGripPose`、`0x17 SetGripPresetPose` 上位机命令
- **速度控制命令**：新增 `0x15 SetMasterChassisVelocity` 车体系速度指令
- **动作诊断模块**：为 Step/Grip 动作增加错误处理和报告逻辑
- **自动化测试**：新增 automapping 脚本化低速自动建图测试流程
- **I2C 管理器**：新增共享 I2C 总线管理器，支持多设备共用 I2C2
- **气压传感器**：集成 XGZP6847D 气压计，实现吸盘物体检测

### 修复
- 修复 I2C 总线错误重启问题
- 修复 grip turn 控制频率问题，改为错频发送
- 修复底盘回程差问题
- 优化自检逻辑
- 修复初始化时序问题
- 修复连接检测拆分后的各种同步问题

### 变更
- **夹取机构重构**：Grip arm/turn 从 DJI 电机更换为 DM 电机 (DM4310 + DM2325)
- **吸盘重构**：重构为双吸盘 device 层架构，更新继电器引脚为 RELAY0~3
- **协议模块拆分**：将 PCProtocol 拆分为 PCProtocol、PCCommandHandler、PCFeedback 三个独立模块
- **动作状态反馈**：新增 16 位 ActionState 打包字段，编码 Step/Lift/Grip/吸盘/红外状态
- **连接状态 I2C 发送**：支持通过 I2C 向外发送连接状态表
- **上下 400mm 台阶**：增加 200mm 和 400mm 台阶的上/下动作
- **Grip 动作组**：新增 SpearGrab(取矛头) 和 KfsStore(卷轴暂存/释放) 高层动作组

## 2026-04

### 新增
- **底盘定位系统**：移植 EKF 定位，支持 JustEncoder / LocEKF 两种定位模式
- **上位机协议**：实现完整的 AA BB 帧协议，含 CRC16 Modbus 校验、时钟对齐、反馈帧
- **台阶动作**：实现上/下台阶动作组状态机
- **底盘控制命令**：新增 `0x13/0x14` Master 目标位姿命令
- **升降机构**：新增独立双前/后侧升降，含校准、轨迹规划与控制
- **Grip 夹取机构**：新增 arm/turn 双轴关节、夹爪控制
- **Arena 内存管理**：全局静态 arena 覆盖 new/delete
- **连接状态管理**：16 位连接位图，统一管理所有硬件在线状态

### 变更
- 升降曲线衔接方式改为曲线衔接
- 升降使用双电机同步轨迹
- 使用 homing trajectory 进行堵转校准
- 通过 cpkg 管理项目依赖
